``` ### **Step 2: Update Your Security Rules** This is a **critical step**. Right now, your database only allows writing to the `subscribers` collection. We need to give permission for you (and only you) to write to a new `posts` collection. 1. Go to your **Firebase Console -> Firestore Database -> Rules** tab. 2. **Replace** the existing rules with the following code. This adds a new rule for blog posts while keeping your old rule for subscribers. ``` rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Rule for subscribers (no change) match /subscribers/{subscriberId} { allow create: if true; allow read, update, delete: if false; } // NEW RULE for blog posts match /posts/{postId} { // Allow anyone in the world to read posts for your public blog. allow read: if true; // ONLY allow write access (create, update, delete) if a user is signed in. // Since you are the only user, only you can manage posts. allow write: if request.auth != null; } } }
